EQUATED
Tech Stack & Infrastructure Guide
Easy to build  •  High quality  •  Near-zero upfront cost

This document defines the complete technology stack for Equated — every tool chosen for three reasons: it is easy to implement, it produces production-quality results, and it costs as close to zero as possible at MVP scale.

1. Stack Selection Principles
Every tool in this stack was evaluated against three criteria:
Ease of implementation — can a single developer ship it fast?
Quality ceiling — will it hold up as the product scales?
Cost at zero revenue — ideally free until first ₹10,000 in revenue

The target is: total monthly infrastructure cost under $0 for MVP, and under $15/month at early traction (up to ~500 daily active users).

2. Full Technology Stack
The table below covers every layer of the system with the chosen tool, the reason for selection, its cost, and a backup option.



3. Layer-by-Layer Breakdown
3.1 Frontend — Next.js 14 + Tailwind + shadcn/ui
Next.js is the single best choice for Equated for one reason: it eliminates the need for a separate backend server in the MVP. API routes live in the same repository as the UI, so one developer can build the entire product without managing two codebases.
App Router enables server components — math-heavy pages render server-side, reducing client JS
Tailwind CSS removes all custom CSS maintenance
shadcn/ui provides pre-built, accessible input, dialog, and card components at zero cost
KaTeX renders LaTeX math expressions in the browser — faster and lighter than MathJax
Deploy to Vercel with one command: git push. Vercel's free tier covers all MVP needs including preview deployments on every PR.
3.2 AI Layer — DeepSeek + Groq + Custom Router
The AI layer is the most cost-sensitive part of the system. The strategy is to use the cheapest model that can handle each task type, and only escalate to a stronger model when required.
DeepSeek R1 / V3: Best-in-class reasoning for STEM at roughly $0.001 per 1,000 tokens — 10x cheaper than GPT-4
Groq (Llama 3.3 70B): Sub-200ms responses on simple tasks. Free tier covers 14,400 requests/day — enough for the entire MVP phase
Custom Router: A lightweight TypeScript middleware (100–200 lines) that classifies each incoming problem and selects the right model

Router logic (simplified):

  classify(problem) → subject + complexity
  if complexity = 'low'  → Groq (free)
  if complexity = 'high' → DeepSeek R1 (~$0.001)
  if type = 'math only'  → SymPy directly (free)

The router is model-agnostic by design. Any model can be swapped in or out by changing a single config object. No vendor lock-in.
3.3 Math Engine — SymPy on FastAPI
Language models should not do arithmetic or symbolic math — they hallucinate. SymPy, a Python computer algebra system, handles all mathematical computation with guaranteed correctness.
Deployed as a minimal FastAPI microservice on Render's free tier
Handles: symbolic algebra, differentiation, integration, equation solving, matrix operations
The AI interprets the problem and generates the SymPy expression; SymPy computes it; the AI explains the steps
# Example SymPy usage
from sympy import symbols, integrate, diff
x = symbols('x')
result = integrate(x**2 + 3*x, x)   # → x³/3 + 3x²/2 + C
Render's free tier spins down after 15 minutes of inactivity. For MVP this is fine — add a keep-alive ping if needed.
3.4 Database & Vector Cache — Supabase
Supabase replaces four separate services in one: Postgres database, authentication, file storage, and vector search (via pgvector). This is the single highest-leverage infrastructure choice in the stack.
Postgres stores users, solve history, credits, and cached solutions
pgvector extension enables semantic similarity search on question embeddings
Supabase Auth handles email OTP and Google OAuth — zero code
Supabase Storage holds uploaded homework images
Free tier: 500MB database, 1GB storage, 50,000 monthly active users
The vector cache alone can eliminate 30–60% of all AI API calls at scale. At MVP this also means faster responses for students asking common questions.
3.5 OCR & Parsing — Tesseract + pix2tex
Students frequently photograph homework problems. The parsing pipeline handles image inputs without any paid third-party API.
Tesseract: Open-source OCR engine, self-hosted in the Python microservice
pix2tex: Open-source model that converts images of equations directly to LaTeX
Both run inside the existing Render microservice — no additional hosting cost
Mathpix is the paid alternative ($0.004/call) and produces better results on complex diagrams. It can be added as a premium tier feature later.
3.6 Payments — Razorpay
Razorpay is the best payment gateway for an INR-first product:
No monthly or setup fee — only 1.8% + GST per transaction
Instant settlement to Indian bank accounts
UPI, cards, net banking, and wallets all supported
React SDK available — credit pack purchases can be implemented in an afternoon
The platform only incurs payment fees when revenue is actually flowing. There is no cost during the zero-revenue MVP phase.
3.7 Analytics & Monitoring — Posthog + Sentry
Posthog: Product analytics, user funnels, feature flags, session recordings — all free up to 1 million events/month
Sentry: Error tracking and AI response failure monitoring — 5,000 errors/month free
Together these two cover all observability needs at zero cost for the MVP

5. Cost Breakdown by Service


5.1 Estimated Monthly Cost — MVP Phase (0–500 DAU)
Infrastructure: $0/month (everything on free tiers)
AI API (no cache): ~$0.001 × 500 solves/day × 30 = $15/month worst case
AI API (with 50% cache hit rate): ~$7.50/month
Payment processing: 0% until first credit pack sale
Total estimated monthly cost at MVP: $0 – $8 depending on cache performance.
At 500 daily active users generating 2,500 solves/day with a 60% cache rate: ~$9/month. The free tier revenue buffer from Groq alone eliminates the first 14,400 requests every day.
5.2 Break-Even Analysis
Target cost per solve: $0.0005 – $0.001
Credit pack price: ₹10 for 30 solves ≈ ₹0.33/solve ≈ ~$0.004/solve
Gross margin per paid solve: ~75–80%
Break-even volume: ~20 credit pack purchases per month covers all infrastructure

6. Developer Setup — Getting Started
The entire stack can be running locally in under 2 hours:

# 1. Clone and install
npx create-next-app@latest equated --typescript --tailwind
cd equated
npx shadcn-ui@latest init

# 2. Set up Supabase
#    → Create project at supabase.com (free)
#    → Enable pgvector extension in SQL editor
#    → Copy API keys to .env.local

# 3. Set up Python microservice
pip install fastapi sympy pytesseract pix2tex uvicorn
uvicorn main:app --reload

# 4. Add AI keys to .env.local
DEEPSEEK_API_KEY=...
GROQ_API_KEY=...

# 5. Deploy
git push  # → Vercel auto-deploys frontend
#          → Render auto-deploys Python service


7. Upgrade Path as the Product Scales
Every free-tier service has a clear, affordable upgrade path when traffic grows:
Supabase → Pro ($25/mo): Unlimited DB size, no spin-down, daily backups
Render → Starter ($7/mo): Always-on Python microservice, no cold starts
Vercel → Pro ($20/mo): Higher bandwidth limits, team features
Pinecone → replace pgvector: If vector search becomes a bottleneck at 10M+ embeddings
DeepSeek → add Claude/GPT-4 tier: For premium users who need stronger reasoning
Mathpix → replace Tesseract: For better equation OCR as a paid feature
None of these upgrades require architectural changes. The system is designed so each component can be swapped independently.

8. Stack Summary
The complete Equated stack in one line per layer:
UI: Next.js 14 + Tailwind + shadcn/ui + KaTeX → deployed on Vercel
API: Next.js API Routes (no separate server for MVP)
AI Router: Custom TypeScript middleware — DeepSeek for hard problems, Groq for easy ones
Math: SymPy on FastAPI → deployed on Render free tier
OCR: Tesseract + pix2tex inside the Python service
DB + Auth + Storage + Vector Cache: Supabase (single service, free tier)
Payments: Razorpay (no monthly fee)
Analytics: Posthog + Sentry (both free at MVP scale)

Total services to manage: 4 (Vercel, Render, Supabase, DeepSeek/Groq). Everything else is a library or SDK inside one of these four.
